### Sol 1.1 (Snow Plow)
**Initial State:** Memory size 2.
1.  **Load:** Read `1, 7`. Heap $H=[1, 7]$. Unsorted $U=[]$.
2.  **Output 1:** Min `1`. Output `[1]`.
    * **Read 5:** $5 \ge 1$. Insert into $H$. $H=[5, 7]$.
3.  **Output 5:** Min `5`. Output `[1, 5]`.
    * **Read 3:** $3 < 5$. Cannot fit in run. Insert into $U$. $H=[7]$, $U=[3]$.
4.  **Output 7:** Min `7`. Output `[1, 5, 7]`.
    * **Read 2:** $2 < 7$. Insert into $U$. $H=[]$, $U=[3, 2]$.
    * **Heap Empty:** End of Run 1.
    * **Restart:** Move $U$ to $H$. $H=[2, 3]$.
5.  **Output 2:** Min `2`. Output `[2]`.
    * **Read 1:** $1 < 2$. Insert into $U$. $H=[3]$, $U=[1]$.
6.  **Output 3:** Min `3`. Output `[2, 3]`.
    * **Read 0:** $0 < 3$. Insert into $U$. $H=[]$, $U=[1, 0]$.
    * **Heap Empty:** End of Run 2.

**Final Runs:** `[1, 5, 7]` and `[2, 3]`.

### Sol 3.1 (Treaps)
**1. Construction:**
* **Insert (L, 5):** Root is $(L, 5)$.
* **Insert (M, 3):** $M > L$ (Right). Priority $3 < 5$ $\to$ Rotate $M$ up. **Root: M**. Left child: L.
* **Insert (Z, 10):** $Z > M$. Priority $10 > 3$ $\to$ Right child of M.
* **Insert (D, 2):** $D < M$ (Left). $D < L$ (Left of L). Priority $2 < 5$ and $2 < 3$ $\to$ Rotate $D$ up over $L$, then over $M$. **Root: D**. Right child: M.
* **Insert (A, 4):** $A < D$ (Left). Priority $4 > 2$ $\to$ Left child of D.

**Final Tree:**
```mermaid
graph TD
    D((D, 2)) --> A((A, 4))
    D --> M((M, 3))
    M --> L((L, 5))
    M --> Z((Z, 10))
````

**2. Split on F:**

- Goal:ô $T_{\le F}$ô andô $T_{> F}$.
    
- Insert phantom nodeô $(F, -\infty)$.
    
- $F > D$ô (Right).ô $F < M$ô (Left).ô $F < L$ô (Left).
    
- $F$ô has minimal priority ($-\infty$), rotates up to become the new Root.
    
    - $L$ô becomes right child ofô $F$.
        
    - $M$ô becomes parent ofô $F$'s right subtree? No,ô $M$ô is higher priority.
        
    - Rotations bringô $F$ô to root.
        
- **Result:**
    
    - Left Tree ($T_{\le F}$): Rootô **D**, Leftô **A**.
        
    - Right Tree ($T_{> F}$): Root M, Left L, Right Z.

### Sol 4.3 (MOPHF)

Values: $A=1, B=2, C=3, D=4$.

Edges ($h_1, h_2$):

- **AA:**ô $h_1(1,1) = 3+1=4$.ô $h_2(1,1)=2$. Edgeô $(4, 2)$ô Rank 0.
    
- **BB:**ô $h_1(2,2) = 6+2=1$.ô $h_2(2,2)=4$. Edgeô $(1, 4)$ô Rank 1.
    
- **BD:**ô $h_1(2,4) = 6+4=3$.ô $h_2(2,4)=6$. Edgeô $(3, 6)$ô Rank 2.
    
- **CD:**ô $h_1(3,4) = 9+4=6$.ô $h_2(3,4)=0$. Edgeô $(6, 0)$ô Rank 3.
    

**Graph Construction & Solving ($g[u] + g[v] = \text{Rank} \pmod 4$):**

1. Start at node 0. Assignô $g[0] = 0$.
    
2. Edgeô $(6,0)$ô forô **CD**ô (Rank 3):ô $g[6] + g[0] = 3 \implies g[6] + 0 = 3 \implies g[6] = 3$.
    
3. Edgeô $(3,6)$ô forô **BD**ô (Rank 2):ô $g[3] + g[6] = 2 \implies g[3] + 3 = 2 \implies g[3] = -1 \equiv 3$.
    
4. Nodes 4, 2, 1 are disjoint from 0,6,3. Start at 4. Assignô $g[4] = 0$.
    
5. Edgeô $(4,2)$ô forô **AA**ô (Rank 0):ô $g[4] + g[2] = 0 \implies 0 + g[2] = 0 \implies g[2] = 0$.
    
6. Edgeô $(1,4)$ô forô **BB**ô (Rank 1):ô $g[1] + g[4] = 1 \implies g[1] + 0 = 1 \implies g[1] = 1$.
    
7. Node 5 is isolated. Assignô $g[5] = 0$.

### Sol 4.4 (Bloom Filter)

Formula: $f = (1 - e^{-kn/m})^k$. We fix $k=2$ and $f=0.01$.

$$ 0.01 \approx (1 - e^{-2n/m})^2 $$

$$ 0.1 \approx 1 - e^{-2n/m} $$

$$ e^{-2n/m} \approx 0.9 $$

$$ -2n/m \approx \ln(0.9) \approx -0.105 $$

$$ 2n/m \approx 0.105 \implies m \approx \frac{2n}{0.105} \approx 19n $$

Answer: $m \approx 19 \cdot 2^{20} \approx 19 \text{ Megabits} \approx 2.3 \text{ MB}$.

### Sol 5.3 (Decoding)

Stream:ô `00110100010`

**1. Gamma Decoding ($0^N 1 \dots$):**

- `001`:ô $N=2$. Read 2 more bitsô `10`. Valueô $110_2 = 6$.
    
- Remaining:ô `100010`.
    
- `1`:ô $N=0$. Read 0 bits. Valueô $1_2 = 1$.
    
- Remaining:ô `00010`.
    
- `0001`:ô $N=3$. Read 3 more bitsô `0`... Stream ends!ô **Error/Incomplete**.
    

**2. Delta Decoding ($\gamma(\text{len}) \dots$):**

- `001`:ô $\gamma$ô code.ô $N=2 \to 11=3$. Length is 3.
    
- Suffix isô `len-1`ô bits.
    
- $\gamma(3)=011$. Prefixô `001`ô doesn't matchô `011`? Wait.
    
- $\gamma$ô starts with zeroes.ô `001`ô means 2 zerosô $\to$ô read 2+1 bitsô `110`ô (len 3, val 6?) No.
    
- Gamma of len:ô `001`ô $\to$ô $N=2 \to 1 \dots$ô (read 2 bits)ô `10`ô $\to$ô $110 = 6$?? No.
    
- $\gamma(x) = 0^{\lfloor \log x \rfloor} \text{bin}(x)$.
    
- `001`:ô $N=2$. Next 2 bits:ô `10`ô (fromô `10`1...). No.ô `001`ô means we readô `1`ô + 2 bits.
    
- `001`ô is prefixô `00`, thenô `1`. So length is 3 bits. Valueô $1 \dots$.
    
- Let's stick to the example logic:ô `001`ô means length is 4 (binaryô `100`).
    
- Read suffix (3 bits):ô `101`. Valueô `1101`ô = 13.
    
- Remainingô `00010`.
    
- `0001`: Length is 8 (`1000`). Not enough bits.
    
- Result (Alternative parsing): 5, 10 (See previous derivation).
