Succinct data structures aim to store data using space close to the information-theoretic lower bound while supporting operations efficiently. The fundamental building blocks are **Rank** and **Select** on bitvectors.

## 1. Problem Definition

Given a binary array $B[1 \dots n]$ of length $n$.
* **Rank$_b(i)$**: Returns the number of bits equal to $b$ in the prefix $B[1 \dots i]$.
    * Note: $\text{Rank}_0(i) = i - \text{Rank}_1(i)$. We only need to implement $\text{Rank}_1$.
* **Select$_b(j)$**: Returns the position of the $j$-th occurrence of bit $b$.

### 1.1 Naive Solutions
Let $m$ be the number of 1s in $B$.
1.  **Array of Positions:** Store indices of all 1s.
    * Space: $m \log n$ bits.
    * Select: $O(1)$ (Array access).
    * Rank: $O(\log m)$ (Binary Search).
2.  **Prefix Sums:** Store cumulative count for every position.
    * Space: $n \log n$ bits. (Huge! $32n$ or $64n$).
    * Rank: $O(1)$.
    * Select: $O(\log n)$ (Binary Search).

---

## 2. Succinct Solution ($n + o(n)$ bits)
We keep the original bitvector $B$ ($n$ bits) and add auxiliary structures taking sub-linear space $o(n)$ to support $O(1)$ operations.

### 2.1 Two-Level Directory (Jacobson's Structure)
We divide $B$ into blocks.

1.  **Superblocks (Size $Z$):**
    * Store **Absolute Rank**: The total number of 1s before the start of the superblock.
    * Size: $Z = (\log n)^2$.
    * Space: $(n/Z) \times \log n = n / \log n$ bits ($o(n)$).

2.  **Blocks (Size $z$):**
    * Store **Relative Rank**: The number of 1s from the start of the superblock to the start of this block.
    * Size: $z = \frac{1}{2} \log n$.
    * Space: $(n/z) \times \log Z = (2n / \log n) \times \log((\log n)^2) = O(n \frac{\log \log n}{\log n})$ bits ($o(n)$).

3.  **In-Block Query:**
    * To solve Rank within a small block $z$, we scan.
    * Since $z$ is tiny (e.g., 32 or 64 bits), it fits in a CPU register.
    * **Modern CPU:** Use `POPCOUNT` instruction to count 1s in $O(1)$.
    * **Theoretical:** Use the **Four Russians Trick**. Precompute a table $T$ of all $2^z$ bit patterns.
        * $T[\text{pattern}][\text{offset}]$ stores the rank.
        * Table size is negligible ($2^{\log n/2} = \sqrt{n}$).

### 2.2 Query Algorithm
To compute $\text{Rank}_1(i)$:
1.  **Superblock Index:** $S = \lfloor i / Z \rfloor$. Fetch `Super[S]`.
2.  **Block Index:** $b = \lfloor i / z \rfloor$. Fetch `Block[b]`.
3.  **In-Block:** Scan bits from start of block $b$ to offset $i \pmod z$.
    $$
    \text{Rank}(i) = \text{Super}[S] + \text{Block}[b] + \text{PopCount}(B[\dots])
    $$
* **Total Time:** $O(1)$.
* **Total Space:** $n + o(n)$ bits.

### 2.3 Example Trace
$B = [010111010101110001]$. $n=18$.
* **Superblocks ($Z=6$):**
    * $SB_0$ (Pos 0): 0.
    * $SB_1$ (Pos 6): $\text{pop}(010111) = 4$.
    * $SB_2$ (Pos 12): $4 + \text{pop}(010101) = 4+3 = 7$.
    * $SB_3$: $7 + 3 = 10$.
* **Blocks ($z=3$):**
    * $b_0$ (in $SB_0$): 0.
    * $b_1$ (in $SB_0$): $\text{pop}(010) = 1$.
    * ...
* **Query $\text{Rank}_1(11)$:**
    * Index 11 (0-based? Let's assume 1-based 11th bit).
    * Superblock: $\lfloor 11/6 \rfloor = 1$ (Indices 0,1). Use $SB_1 = 4$.
    * Block in $SB_1$: Offset $11 - 6 = 5$. $\lfloor 5/3 \rfloor = 1$. Use $b_1 = 1$ (Relative to SB).
    * In-Block: Offset $5 \pmod 3 = 2$. Bits `010` (from pos 9,10,11).
    * Scan first 2 bits of `010`? No, offset 2 means 3rd bit.
    * Let's assume bits at 9,10,11 are `0,1,0`? No, $B$ is `010101...`
    * Bits in block: `101`?
    * **Total:** $4 + 1 + 1 = 6$.

## 3. Compressed Solution (Elias-Fano)
If the bitvector is sparse (few 1s), we can do better than $n$ bits.
We store the **positions of the 1s** as an increasing sequence $S$ using **Elias-Fano**.

### 3.1 Operations
* **Select$_1(i)$**: This is exactly **Access($i$)** on the EF-encoded sequence.
    * Returns the position of the $i$-th 1.
    * **Time:** $O(1)$.
* **Rank$_1(j)$**: This is finding the index $i$ such that $S[i] \le j < S[i+1]$.
    * Equivalent to **NextGEQ($j+1$)**. The result $x$ is the value $\ge j+1$.
    * The rank is the index of the value before it.
    * **Time:** $O(\log(u/m))$.

### 3.2 Recursive Definition
Elias-Fano itself relies on a bitvector $H$ (unary high parts). To allow $O(1)$ operations on EF, we need Rank/Select on $H$.
* Since $H$ is dense (or we just need succinctness), we use the **Succinct ($n+o(n)$)** structure for $H$.
* **Stack:** Compressed Problem $\to$ Elias-Fano $\to$ Succinct Bitvector on $H$.
